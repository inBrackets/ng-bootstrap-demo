Class & Style Bindings

Angular lets you bind CSS classes and inline styles directly to component state — no manual DOM manipulation needed. Brackets [...] mean "evaluate this as JavaScript and use the result."

1 — Single class: [class.highlighted]="isHighlighted()"

When the expression is true the class is added; when false it is removed. Angular handles the DOM update for you.

{{ isHighlighted() ? 'I am highlighted!' : 'I have no highlight.' }}
isHighlighted() = {{ isHighlighted() }}
2 — Object syntax: [class]="alertClasses()"

Pass an object — Angular adds classes whose values are true and removes the rest. Great when several classes are mutually exclusive (like alert colours).

Alert type is: {{ alertType() }}
3 — Multiple classes from checkboxes

Each checkbox controls one signal. A computed() builds the class object from all three — Angular re-evaluates it whenever any signal changes.

The quick brown fox jumps over the lazy dog.

4 — Single style: [style.color]="textColor()"

[style.color] sets only the color property. Angular converts the value to style="color: …" on the element.

This text colour is controlled by a signal.

textColor() = "{{ textColor() }}"
5 — Style with unit: [style.font-size.px]="fontSize()"

Adding .px (or .%, .em etc.) after the property name means you store a plain number in your signal — no string concatenation needed.

Drag the slider to resize me!

{{ fontSize() }}px
6 — Object syntax: [style]="boxStyle()"

Return a plain object from computed(). Angular applies every property in it directly to the element's style attribute.

My background, padding and border-radius are all style-bound.
7 — Class & style together: a live status card

A single element can have both [class] and [style] at the same time. Computed signals keep both in sync with one shared piece of state (cardStatus).
