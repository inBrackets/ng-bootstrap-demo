Control Flow: @if, @for, @switch

Angular 17 replaced structural directives (*ngIf, *ngFor, *ngSwitch) with built-in block syntax. The new blocks are faster, require no imports, and support extra features like @empty and the as alias.

@if — conditional rendering
1 — @if / @else

Angular renders the @if block when the expression is truthy, and the @else block when it is falsy. The other block is never created in the DOM.

@if (showAlert()) {
This is rendered by the @if block.
} @else {
This is rendered by the @else block.
}
showAlert() = {{ showAlert() }}
2 — @if / @else if / @else

Chain as many @else if branches as you need. Angular evaluates them top-to-bottom and renders only the first match.

@if (score() >= 90) {
A Excellent — score ≥ 90
} @else if (score() >= 80) {
B Good — score ≥ 80
} @else if (score() >= 70) {
C Average — score ≥ 70
} @else if (score() >= 60) {
D Below average — score ≥ 60
} @else {
F Failing — score < 60
}
score() = {{ score() }}
3 — @if (profile(); as p) — the as alias

The as keyword binds the truthy result to a local variable. Inside the block p is typed as the non-null version of the expression — no optional chaining needed. The @else block renders when profile() is null.

@if (profile(); as p) {
{{ p.avatar }}
{{ p.name }}
{{ p.role }}
} @else {

No profile loaded

}

With as p (recommended):

  • profile() is called once
  • p is typed as non-null inside the block
  • No !. non-null assertions needed

Without as:

  • You'd call profile() separately for each property
  • TypeScript still thinks it could be null
  • You'd need profile()!.name or optional chaining everywhere
@for — list rendering
4 — Loop context variables: $index $first $last $even $odd $count

Inside every @for block Angular exposes six implicit variables — no imports or declarations required. Use them directly in the template. track is required: it tells Angular how to identify each item so it can update only the changed rows instead of re-rendering the whole list. For primitives use track item; for objects use track item.id.

Item $index $first $last $even $odd $count
@for (color of colors; track color) {
{{ color }} {{ $index }} {{ $first }} {{ $last }} {{ $even }} {{ $odd }} {{ $count }}
}
5 — @for + @empty

@empty renders automatically when the iterable is empty — no *ngIf="items.length === 0" needed. Remove all tasks to see it.

@for (task of tasks(); track task.id) {
{{ task.text }}
} @empty {

No tasks — add one below.

}
{{ tasks().length }} items
6 — Nested @for

@for blocks can be nested. Each level has its own independent $index, $first etc. — they don't interfere with each other.

@for (group of techStack; track group.category) {
{{ group.category }} {{ group.items.length }}
@for (skill of group.items; track skill) {
{{ skill }} @if ($last) { last }
}
}
@switch — value switching
7 — @switch / @case / @default

@switch is cleaner than a long @if / @else if chain when you're matching one expression against several fixed values. Comparison is always ===. There is no fall-through between cases.

@switch (orderStatus()) { @case ('pending') {
Pending

Your order is waiting to be confirmed.

} @case ('processing') {
Processing

We are preparing your package.

} @case ('shipped') {
Shipped

Your package is on its way!

} @case ('delivered') {
Delivered

Package delivered. Enjoy!

} @default {
Cancelled

This order has been cancelled.

} }
@for (s of statuses; track s) { }
8 — @switch on a number

@switch works on any value type — strings, numbers, booleans. Here a number signal drives both a star display (using @for + @if) and a feedback message (using @switch).

@for (n of [1,2,3,4,5]; track n) { @if (n <= rating()) { } @else { } }
@switch (rating()) { @case (1) {
Needs significant improvement
} @case (2) {
Below expectations
} @case (3) {
Meets expectations
} @case (4) {
Exceeds expectations
} @case (5) {
Outstanding!
} } rating() = {{ rating() }}
