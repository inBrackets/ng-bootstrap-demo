Deferrable Views @defer

@defer lazy-loads a piece of your template — and all the components inside it — only when it's actually needed. The browser skips downloading, parsing and compiling that JavaScript until the trigger fires, speeding up the initial page load.

Every @defer block has three optional companion blocks that control what the user sees at each stage of the lifecycle:

@placeholder Shown immediately. Removed when the trigger fires.
→
@loading Shown while content is being downloaded.
→
@defer content Rendered once loading is complete.
→
@error Rendered if downloading failed.
1 — @defer (on interaction)

Trigger: the user clicks or presses a key on the placeholder. Nothing is downloaded until that moment.

@defer (on interaction) {
on interaction
Content loaded!

Angular downloaded and rendered this block only after you interacted with the placeholder. In a real app you would put a heavy component (chart, editor, map…) here.

} @placeholder {

Click here to load

(@placeholder block)

} @loading (minimum 800ms) {

Loading… (@loading block)

} @error {
Failed to load content. (@error block)
}
2 — @defer (on hover)

Trigger: the cursor moves over the placeholder. Great for tooltips, preview panels, or any secondary info the user might not need.

@defer (on hover) {
on hover
Hover content loaded!

The download started the moment your cursor entered the placeholder. You can pair this with prefetch on idle to start the download even earlier (see the Prefetch card below).

} @placeholder {

Hover to load

(@placeholder block)

} @loading (minimum 800ms) {

Loading…

}
3 — @defer (on timer(4s))

Trigger: a fixed delay after the component first renders. Useful for non-critical widgets (chat bubbles, cookie banners) that should not slow down the initial paint.

@defer (on timer(4s)) {
on timer
Timer fired!

This content loaded 4 seconds after the component was initialised — no user action required. Navigate away and back to reset the timer and watch it again.

} @placeholder {

Loading in 4 s…

(@placeholder block)

} @loading (after 100ms; minimum 700ms) {

Loading…

}
4 — @defer (when loadOnDemand())

Trigger: a JavaScript expression becomes true. You control exactly when loading starts — e.g. after the user logs in, a feature flag is enabled, or a wizard step is complete.

@defer (when loadOnDemand()) {
when condition
Condition was met!

loadOnDemand() returned true so Angular loaded this block. Setting it back to false won't unload the content — @defer is one-way.

} @placeholder {

Waiting for condition…

(@placeholder block)

} @loading (minimum 800ms) {

Loading…

}

5 — @defer (on viewport)

Trigger: the placeholder element enters the browser's visible viewport (uses IntersectionObserver under the hood). This is the highest-impact trigger — content below the fold is never downloaded until the user actually scrolls to it.

@defer (on viewport) {
on viewport
You scrolled here — content loaded!

Angular used IntersectionObserver to detect that the placeholder entered the viewport, then downloaded and rendered this block. Users who never scroll here never download this JavaScript at all.

} @placeholder {

Scroll this into view to load

Angular monitors this element with IntersectionObserver. The moment it enters the viewport the download starts. (@placeholder block)

} @loading (after 100ms; minimum 700ms) {

Loading below-the-fold content…

}
6 — prefetch: separate "download" from "render"

Every trigger can have a companion prefetch trigger. Prefetch starts the download early while the main trigger still controls when the content appears. The result: near-instant rendering when the main trigger fires because the JS is already cached in memory.

@defer (on interaction;
  prefetch on idle)

Download while the browser is idle, render on click. Best for content that most users will eventually open.

@defer (on interaction;
  prefetch on hover)

Download when the cursor approaches, render on click. Removes perceived latency for mouse users.

@defer (on viewport;
  prefetch on idle)

Download at idle, render only when scrolled to. Content appears instantly when the user reaches it.
